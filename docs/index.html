# Ramidus Docs Ramidus is a SPA framework built with, and designed for, custom elements. ## Table of Contents - [Getting Started]() - [Project Structure]() - [Pages]() - [Layouts]() - [Components]() - [Deployment]() ## Getting Started Setup is easy and no tooling is required. You can scaffold your project using `npx` or simply download [this zip file](https://codeload.github.com/jameslovallo/ramidus/zip/refs/heads/main). 
npx ramidus@latest && npm run dev
## Project Structure Ramidus is a runtime framework and is therefore structured to use 100% vanilla routing, i.e. `/docs/index.html`. The "@" folder contains everything relevant to your application like components, assets and layouts. "@" is short for "app." This convention keeps the app folder first alphabetically so you don't lose it in your content.
  • @
    • assets
    • components
    • css
    • build.js
    • main.js
    • head.json
  • docs
    • index.html
  • index.html
### @ Folder Contents | item | description | | ----------- | ------------------------------------------- | | /assets | images, icons, and other assets | | /components | custom element source code | | /css | global stylesheets | | build.js | called by `npm run build` during deployment | | main.js | register custom elements here | | head.json | anything that goes in the document head | ## Pages Writing pages in Ramidus is just like writing pages in a vanilla site, with one small exception: Ramidus has templates! There are two things you must include in every page to make this work. 1. A script tag loading `/@/main.js` 2. A body tag with the `is` attribute loading an appropriate template element. The next section will address how to create templates. 
<script src="/@/main.js" type="module"></script>

<body is="app-layout">...</body>
You may also write pages in markdown by adding `lang="md"` to the body. 
<body lang="md" is="app-layout">...</body>
